# コードレビュー

handson/ ディレクトリのコードをレビューします。

## 対象ファイル

- handson/src/*.ts
- handson/test/*.test.ts

## レビュー観点

### 1. TDD 準拠

以下を確認:
- [ ] テストが先に書かれているか（コミット履歴から推測）
- [ ] テストカバレッジは十分か
- [ ] エッジケースがカバーされているか
- [ ] テストケースの命名は適切か

### 2. コード品質

以下を確認:
- [ ] 命名規則に従っているか（camelCase, PascalCase）
- [ ] 関数は単一責務か
- [ ] エラーハンドリングは適切か
- [ ] イミュータブルな実装になっているか

### 3. TypeScript

以下を確認:
- [ ] 型定義は適切か
- [ ] any 型を使っていないか
- [ ] 型推論が効いているか
- [ ] interface/type の使い分けは適切か

### 4. ドキュメント

以下を確認:
- [ ] JSDoc コメントがあるか
- [ ] 複雑なロジックに説明があるか
- [ ] 例外がスローされる条件が明記されているか

## 優先度

改善提案には優先度を付与:
- 🔴 **高**: すぐに修正すべき
- 🟡 **中**: 次回リリースまでに修正
- 🟢 **低**: 時間があれば対応
