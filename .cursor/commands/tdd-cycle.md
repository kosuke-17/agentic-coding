# TDD サイクル実行

指定された機能を TDD で実装します。

## 入力情報

ユーザーから以下の情報を受け取ります：
- 機能名（実装する関数名）
- ファイル名（テストと実装のベースファイル名）

## 手順

### 1. 🔴 Red フェーズ

handson/test/{ファイル名}.test.ts にテストを追加:

1. 機能の仕様を確認
2. テストケースを洗い出す
   - 正常系
   - 異常系
   - エッジケース
3. テストコードを記述
4. テストを実行して **失敗** を確認

```bash
pnpm test handson/test/{ファイル名}.test.ts
```

### 2. 🟢 Green フェーズ

handson/src/{ファイル名}.ts に実装を追加:

1. テストが通る **最小限** のコードを実装
2. 過度な最適化は行わない
3. テストを実行して **成功** を確認

```bash
pnpm test handson/test/{ファイル名}.test.ts
```

### 3. 🔵 Refactor フェーズ

コードを改善:

1. 重複を排除
2. 変数名・関数名を改善
3. コメント/JSDoc を追加
4. テストが引き続き **成功** することを確認

```bash
pnpm test handson/test/{ファイル名}.test.ts
```

## 報告

各フェーズの結果を以下の形式で報告してください：

### 🔴 Red フェーズ結果
- 作成したテストケース
- テスト実行結果（失敗を確認）

### 🟢 Green フェーズ結果
- 実装した内容
- テスト実行結果（成功を確認）

### 🔵 Refactor フェーズ結果
- 改善した内容
- テスト実行結果（成功を維持）
